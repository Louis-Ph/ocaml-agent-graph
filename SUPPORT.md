# Support

## Where to ask for help

- bug reports: open a GitHub issue with clear reproduction steps
- feature requests: open a GitHub issue describing the operator or developer problem first
- security issues: follow [SECURITY.md](SECURITY.md) and do not report them publicly

## What to include

When asking for help, include:

- the commit or release you are using
- the relevant graph config or demo pack
- the relevant `route_model` or provider family when LLM behavior is involved
- sanitized config snippets
- sanitized logs or terminal output
- the exact command or input that reproduces the problem

## Support expectations

This is currently an early-stage project. Support is best-effort and review
bandwidth may vary.

Questions that are small, reproducible, and precise will be resolved faster
than broad architecture requests with no concrete configuration context.
